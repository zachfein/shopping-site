Gotchas

The link to the melon image is finicky. Once fixed, it might still be broken on the melon details page. This solution seems to not be always correct, but at first blush, static should always be referenced from the root of the server, ex: src="/static/img/watermelon.png"

Flask can’t detect changes to mutable objects in session. One way to make this work is to manually set session.modified = True or to add a flash message (which is pretty hacky but oh well)

Basically, make sure students follow the instructions and add a flash message before redirecting (if they’re ever attempting to alter session)

Flask is fine handling assignments and deletions. This behavior just causes problems when students attempt to add new melons to their cart. So make sure they do the flash message thing or set session.modified manually
